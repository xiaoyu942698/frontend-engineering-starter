# 可访问性规范

可访问性要求以不增加复杂度为前提，优先保证后台和智能体工作流中的关键操作可理解、可操作。

## 基础规则

- 按钮必须有可读名称，图标按钮必须有 `aria-label` 或 tooltip。
- 表单项必须有关联 label、错误提示和帮助文本。
- 弹窗和抽屉必须有明确标题。
- 危险操作必须有确认文案。
- 颜色不能作为唯一状态表达。
- 链接和按钮语义不能混用。

## 键盘与焦点

- 常用操作必须可键盘触达。
- 弹窗打开后焦点应进入弹窗，关闭后回到触发点。
- 不移除默认 focus outline，除非提供等价 token 化样式。
- 自定义可点击区域需要键盘事件和语义角色。

## 智能体场景

- 运行日志需要能按时间顺序阅读。
- 审批操作必须区分批准、拒绝和取消。
- 工具调用详情不能只靠颜色表达成功或失败。
- 长输出需要可复制、可折叠或明确截断。

## 验收

- UI 变更至少人工检查桌面和移动视口。
- 复杂表单、弹窗、审批流需要测试焦点和错误提示。
- 后续可接入 axe 或 Playwright 可访问性扫描作为 hard gate。
