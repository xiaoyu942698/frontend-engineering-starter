# 性能预算规范

本模板默认服务后台和智能体系统，性能目标是稳定、可扫描、可持续迭代。

## 当前阶段

当前 build 会保留 Vite chunk warning。新项目接入真实路由后，应逐步把性能预算从 warning 提升为 hard gate。

## 预算建议

- 首屏关键 route chunk 建议小于 350KB gzip。
- 单个业务 route chunk 建议小于 180KB gzip。
- 公共 vendor chunk 超过 500KB gzip 时必须评估拆分。
- 首屏请求数量和大数据表格渲染需要有明确策略。

## 规则

- 页面路由默认懒加载。
- 重型图表、编辑器、流程画布、富文本、地图等能力按路由或组件动态加载。
- 表格超过 200 行必须分页、服务端分页或虚拟滚动。
- 长列表必须有 loading、empty、error 和分页/继续加载策略。
- 不为装饰引入大体积动画、图表或 3D 依赖。

## Review 重点

- 是否因为新页面把首包显著放大。
- 是否把只在少数页面使用的库放进全局入口。
- 是否对大列表、大日志、事件流做了增量渲染或截断。
- 是否有可操作的 build 输出或 bundle 分析证据。
