# 前端安全规范

前端安全目标是减少泄露、误展示和绕过风险。前端权限只负责体验和入口控制，后端必须二次校验。

## 密钥与环境

- 不提交真实 `.env`、token、cookie、私有 endpoint、生产账号或密钥。
- Vite 环境变量必须以 `VITE_` 开头。
- 组件不能直接读取 `import.meta.env`，必须通过 `shared/config/env`。
- 不在日志、错误信息或 UI 中展示敏感 token。

## 鉴权与权限

- 路由权限走 route meta 和 router guard。
- 操作级权限走 `v-permission` 或 auth store。
- 危险操作必须二次确认。
- 前端不能用隐藏按钮替代后端权限校验。

## XSS 与 HTML

- 禁止随意使用 `v-html`。
- 必须渲染富文本时，需要明确 sanitization 策略。
- 用户输入、模型输出、工具输出默认按纯文本展示。
- Agent 产物展示必须按 artifact 类型选择安全渲染方式。

## 文件与外链

- 上传必须限制类型、大小和错误提示。
- 下载必须显示文件名、来源和风险状态。
- 外链使用新窗口时需要 `rel="noopener noreferrer"`。
- 不自动打开未验证链接。

## 依赖

- 不引入不维护或用途不清的依赖。
- 新依赖需要说明用途、替代方案和体积影响。
- 安全扫描或 audit 失败需要明确处理结论，不能静默忽略。
