# 文档导航

这份文档汇总仓库内的全部 Markdown 文档，说明每份文档写给谁看、主要解决什么问题，以及在什么情况下需要阅读。

如果不确定从哪里开始，可以按下面的阅读路径选择：

- 想用模板创建项目：先看 `README.md`，再看《给 Codex 创建新项目的说明》。
- 刚加入前端项目：先看《前端工程边界说明》，再看《前端开发规范》和《新前端模块模板》。
- 让 Codex 开发或审查代码：从 `AGENTS.md` 进入，Codex 会按《Codex 规则路由》选择对应规范。
- 维护模板、CI 或发布流程：重点看《团队最终使用说明》《Codex Template Rollout Plan》《Git 提交与合并门禁》和《GitHub 分支保护配置》。

## 维护约定

- 新增、删除、重命名 Markdown 文档时，同步更新本导航。
- 文档职责发生变化时，更新对应的“主要用途”和“什么时候看”。
- 本导航只做索引和概述，不重复复制各规范正文。
- 面向团队的最终使用说明仍以 `docs/TEAM_TEMPLATE_GUIDE.md` 为准；面向非前端人员的 Codex 创建项目说明仍以 `template-init/HUMAN_NEW_PROJECT_GUIDE.md` 为准。
